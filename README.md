# GIT - Introdución
> [!IMPORTANT]
> **Git es un sistema de control de versiones distribuido que permite registrar y gestionar los cambios realizados en archivos a lo largo del tiempo.**
>
> Su **objetivo principal** es **coordinar el trabajo entre las personas que participan en un proyecto**, permitiendoles modificar los mismos archivos sin perder el historial de cambios.

### Características principales

* **Historial de cambios:** guarda todas las versiones de los archivos.

* **Trabajo colaborativo:** varios desarrolladores pueden trabajar simultáneamente.

* **Ramas (branches):** permite desarrollar funcionalidades en paralelo.

* **Sistema distribuido:** cada desarrollador tiene una copia completa del repositorio y del historial.

* **Fusión de cambios (merge):** integra modificaciones realizadas por diferentes usuarios.

### Ejemplo conceptual

Supongamos que tres personas trabajan en el mismo proyecto:
1. Cada uno tiene una copia del repositorio.

2. Realizan cambios en archivos.

3. Registran esos cambios con commits.

4. Luego integran los cambios al repositorio compartido.

**De esta forma Git mantiene el historial y resuelve conflictos si dos personas modifican la misma parte del archivo.**

### Conceptos básicos de Git

* **Repositorio (repository):** proyecto gestionado por Git.

* **Commit:** registro de un cambio en el historial.

* **Branch:** línea de desarrollo independiente.

* **Merge:** unión de cambios entre ramas.

* **Clone:** copia completa de un repositorio.


> [!IMPORTANT]
> **En resumen: Git permite controlar versiones de archivos, mantener historial de cambios y facilitar el trabajo colaborativo en proyectos de software.**

# Acerca de GitHub
**GitHub** es una **plataforma basada en la nube** donde puedes almacenar, compartir y trabajar junto con otros usuarios para escribir código.

Almacenar tu código en un "repositorio" en GitHub te permite lo siguiente:

* Presentar o compartir el trabajo.
* Seguir y administrar los cambios en el código a lo largo del tiempo.
* Dejar que otros usuarios revisen el código y realicen sugerencias para mejorarlo.
* Colaborar en un proyecto compartido, sin preocuparse de que los cambios afectarán al trabajo de los colaboradores antes de que esté listo para integrarlos.
* **El trabajo colaborativo, una de las características fundamentales de GitHub, es posible gracias al software de código abierto Git, en el que se basa GitHub.**
